---
title: "Common Issues"
description: "Solutions for common issues when setting up and using Asymptote."
---

<AccordionGroup>
  <Accordion title="Scans aren't triggering on pull requests">
    - Verify the Asymptote GitHub App is installed on the target repository.
    - Confirm the repository is linked in the Asymptote dashboard.
    - Check that webhook deliveries are succeeding under your GitHub App settings → **Recent Deliveries**.
  </Accordion>
  <Accordion title="Status checks are stuck as pending">
    - Large pull requests may take a few extra minutes to process.
    - Re-link the repository from the Asymptote dashboard if it was originally connected through the legacy GitHub Action flow.
    - Check that the repository still has access to the Asymptote GitHub App.
  </Accordion>
  <Accordion title="Permission errors on the GitHub App">
    Re-install the Asymptote GitHub App and ensure the correct repositories are selected. See the [GitHub integration page](/deployment/github) for a full list of required permissions.
  </Accordion>
  <Accordion title="The CLI isn't authenticating">
    Run `asymptote whoami` to check your current auth state. If unauthenticated, run `asymptote login` to re-authenticate. See the [CLI authentication docs](/cli/login) for more details.
  </Accordion>
  <Accordion title="My coding agent isn't being intercepted">
    Ensure Asymptote is configured for your agent. See the [Coding Agent Integrations](/integrations/cursor) docs for setup instructions for your specific tool.
  </Accordion>
  <Accordion title="How do I disconnect a repository?">
    In GitHub: **Repository → Settings → GitHub Apps → Asymptote Security**, then suspend or uninstall the app.
  </Accordion>
  <Accordion title="How do I migrate from the legacy GitHub Action to the GitHub App?">
    In the Asymptote dashboard, remove the repository, then re-link it through the GitHub App flow. The GitHub App should be onboarded automatically when the repository is linked again.
  </Accordion>
  <Accordion title="How do I disconnect all repositories?">
    In GitHub: **Organization → Settings → GitHub Apps → Asymptote Security**, then suspend or uninstall the app.
  </Accordion>
  <Accordion title="How do I remove agent hooks from my local machine?">
    Use the uninstall command for whichever agent you set up:

    **Cursor**
    ```bash
    asym cursor uninstall          # user-level (all projects)
    asym cursor uninstall --project  # current project only
    ```

    **Claude Code**
    ```bash
    asym claude uninstall          # user-level (all projects)
    asym claude uninstall --project  # current project only
    asym claude uninstall --vscode   # VS Code extension
    ```

    **GitHub Copilot** (project-level only, run from the repo root)
    ```bash
    asym copilot uninstall
    ```

    **Factory**
    ```bash
    asym factory uninstall           # user-level (all projects)
    asym factory uninstall --project  # current project only
    ```

    After uninstalling Cursor hooks, restart Cursor to fully deactivate them. See the [integrations docs](/integrations/cursor) for your specific tool.
  </Accordion>
</AccordionGroup>
