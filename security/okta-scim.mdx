# Okta SCIM

Configure SCIM (System for Cross-domain Identity Management) to automatically provision and deprovision users in Asymptote.

## Overview

SCIM enables automatic synchronization of users and groups between Okta and Asymptote, ensuring:

- Automatic user creation when added to Okta
- Automatic user deactivation when removed from Okta
- Group membership synchronization
- Profile updates in real-time

## Prerequisites

- Okta administrator access
- Asymptote Enterprise plan
- SAML or OIDC SSO already configured

## Setup Steps

### 1. Generate SCIM Token

1. Log in to Asymptote as an admin
2. Navigate to **Settings** → **Authentication** → **SCIM**
3. Click **Generate SCIM Token**
4. Copy the token (you won't be able to see it again)

<Warning>
Store this token securely. It provides access to manage users in your organization.
</Warning>

### 2. Configure Okta Provisioning

1. Go to your Asymptote application in Okta
2. Navigate to the **Provisioning** tab
3. Click **Configure API Integration**
4. Check **Enable API integration**
5. Enter the following:

| Setting | Value |
|---------|-------|
| Base URL | `https://api.asymptotelabs.ai/scim/v2` |
| API Token | Your SCIM token from step 1 |

6. Click **Test API Credentials**
7. Click **Save**

### 3. Configure Provisioning Settings

#### To App

Enable the following:

- ✅ Create Users
- ✅ Update User Attributes
- ✅ Deactivate Users

#### Attribute Mappings

| Okta Attribute | Asymptote Attribute |
|----------------|---------------------|
| userName | userName |
| email | email |
| givenName | name.givenName |
| familyName | name.familyName |
| displayName | displayName |

### 4. Configure Group Push

1. Go to the **Push Groups** tab
2. Click **Push Groups**
3. Select groups to synchronize
4. Choose **Push group memberships immediately**

## Supported Operations

| Operation | Supported |
|-----------|-----------|
| Create User | ✅ |
| Update User | ✅ |
| Deactivate User | ✅ |
| Delete User | ✅ |
| Create Group | ✅ |
| Update Group | ✅ |
| Delete Group | ✅ |
| Push Group Members | ✅ |

## Monitoring

View provisioning logs in Asymptote:

1. Navigate to **Settings** → **Audit Log**
2. Filter by **Source: SCIM**

```json
{
  "timestamp": "2024-01-15T10:30:00Z",
  "source": "SCIM",
  "action": "user.created",
  "user": "newuser@example.com",
  "provider": "Okta"
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Test Connection Failed">
    - Verify the SCIM token is correct
    - Ensure the Base URL is exactly `https://api.asymptotelabs.ai/scim/v2`
  </Accordion>
  <Accordion title="User Not Created">
    - Check Okta provisioning logs for errors
    - Verify attribute mappings are correct
  </Accordion>
  <Accordion title="Groups Not Syncing">
    - Ensure Push Groups is configured
    - Verify group names don't contain special characters
  </Accordion>
</AccordionGroup>
