# Okta SSO

Integrate Asymptote with Okta to enable Single Sign-On (SSO) for your organization.

## Prerequisites

- Okta administrator access
- Asymptote Enterprise plan
- Domain verification completed

## Setup Steps

### 1. Create Okta Application

1. Log in to your Okta Admin Console
2. Navigate to **Applications** → **Applications**
3. Click **Create App Integration**
4. Select **OIDC - OpenID Connect**
5. Select **Web Application**
6. Click **Next**

### 2. Configure Application Settings

Enter the following settings:

| Setting | Value |
|---------|-------|
| App integration name | Asymptote |
| Grant type | Authorization Code |
| Sign-in redirect URI | `https://app.asymptotelabs.ai/auth/okta/callback` |
| Sign-out redirect URI | `https://app.asymptotelabs.ai/auth/logout` |

### 3. Configure Assignments

1. Go to the **Assignments** tab
2. Assign the application to the appropriate users or groups
3. Click **Save**

### 4. Get Credentials

From the **General** tab, copy:
- Client ID
- Client Secret

### 5. Configure Asymptote

1. Log in to Asymptote as an admin
2. Navigate to **Settings** → **Authentication**
3. Select **Okta SSO**
4. Enter your Okta credentials:

```
Client ID: <your-client-id>
Client Secret: <your-client-secret>
Okta Domain: <your-okta-domain>.okta.com
```

5. Click **Save** and **Test Connection**

## Testing

1. Log out of Asymptote
2. Click **Sign in with Okta**
3. Authenticate with Okta
4. Verify successful login

## Troubleshooting

<AccordionGroup>
  <Accordion title="Invalid redirect URI">
    Ensure the redirect URI in Okta matches exactly: `https://app.asymptotelabs.ai/auth/okta/callback`
  </Accordion>
  <Accordion title="User not assigned">
    Verify the user is assigned to the Asymptote application in Okta.
  </Accordion>
  <Accordion title="Invalid credentials">
    Double-check the Client ID and Client Secret.
  </Accordion>
</AccordionGroup>
