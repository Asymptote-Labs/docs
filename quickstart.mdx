## Prerequisites

Before installing Asymptote, you must have:
- A GitHub account with admin access to the repositories you plan to connect
- One or more GitHub repositories where you want Asymptote to help detect and prevent code vulnerabilities

## Installing Asymptote

### Step 1: Sign up for Asymptote <span style={{ color: '#6b7280', fontSize: '0.8em', fontWeight: 'normal' }}>· ~1 min</span>

Visit [Asymptote](https://asymptotelabs.ai/login) to create a new account. You can sign up with Google or GitHub, or use your email address and set a password.

Your account includes access to the Asymptote dashboard, where you can:

- **Define and manage security [policies](/introduction/core-concepts#policy)** that match your team's standards
- **Review detected [vulnerabilities](/introduction/core-concepts#vulnerability)**, including what was fixed automatically and what still needs action
- **Track product security and remediation trends** over time with clear metrics and reporting

### Step 2: Connect Your GitHub Account <span style={{ color: '#6b7280', fontSize: '0.8em', fontWeight: 'normal' }}>· ~1 min</span>
<Steps>
  <Step title="Sign into GitHub">
    Visit [this link](https://github.com/apps/asymptote-security/installations/select_target) to install the Asymptote GitHub App.

    <img src="/public/asymptote_github_connection.png" alt="Asymptote GitHub Connection" style={{ maxWidth: '400px' }} />
  </Step>
  <Step title="Authorize GitHub Integration">
    You'll be redirected to GitHub to authorize Asymptote's access to your repositories. Asymptote requires the following permissions:

    - **Read access** to repository code and metadata
    - **Webhook access** to receive real-time repository updates
    - **Read and write access** to checks and commit statuses, issues and pull requests, and repository webhooks

    <img src="/public/asymptote_permissions.png" alt="Asymptote Permissions" style={{ maxWidth: '400px' }} />
  </Step>
  <Step title="Select Repositories">
    Select which repositories you want Asymptote to connect to. You can:

    - Grant access to all repositories in your organization, or
    - Select specific repositories now

    You can add more repositories later from the Asymptote dashboard if needed.
  </Step>
</Steps>

### Step 3: Install the CLI <span style={{ color: '#6b7280', fontSize: '0.8em', fontWeight: 'normal' }}>· ~30 sec</span>

Install the Asymptote CLI using Homebrew:

```bash
brew install asymptote-labs/tap/asymptote-cli
```

<Accordion title="Example output">
```bash
$ brew install asymptote-labs/tap/asymptote-cli

==> Downloading asymptote-cli...
==> Installing asymptote-cli...
✓ Installed asym v0.5.0
```
</Accordion>

### Step 4: Run `asym init` <span style={{ color: '#6b7280', fontSize: '0.8em', fontWeight: 'normal' }}>· ~30 sec</span>

Run the init command to authenticate and connect your AI coding agents:

```bash
asym init
```

This will log you into your Asymptote account, automatically detect supported [coding agents](/introduction/core-concepts#coding-agent) on your machine (Cursor, Claude Code, GitHub Copilot, and more), and allow you to choose which ones to configure Asymptote [hooks](/introduction/core-concepts#hooks) for.

### Step 5: Configure GitHub Security PR Reviews <span style={{ color: '#6b7280', fontSize: '0.8em', fontWeight: 'normal' }}>· ~1 min</span>

Enable the Asymptote GitHub App to automatically scan pull requests and enforce security [policies](/introduction/core-concepts#policy) before code is merged.

<Steps>
  <Step title="Install the GitHub App">
    From the Asymptote dashboard, open **Settings → GitHub** and click **Install GitHub App**.
  </Step>
  <Step title="Link your repositories">
    Select the repositories you want Asymptote to review. The GitHub App is onboarded automatically for PR reviews when the repository is linked.
  </Step>
  <Step title="Require the check to merge">
    In GitHub, enable **Require status checks to pass before merging** in your branch protection rules and select **Asymptote Security Review**.
  </Step>
</Steps>

Once connected, Asymptote will post inline review comments with AI-generated fix suggestions on every pull request. For migration steps from the legacy GitHub Action flow, see the [GitHub Security PR Reviews](/deployment/ci-github-actions) guide.
