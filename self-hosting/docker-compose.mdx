## Overview

This guide covers deploying Asymptote using Docker Compose on a single Linux VM. Choose between automated AWS provisioning with Terraform or manual setup for other environments.

## AWS with Terraform

The fastest way to get Asymptote running on AWS.

### Prerequisites

- AWS account with appropriate permissions
- [Terraform](https://terraform.io) installed (v1.0+)
- AWS CLI configured with credentials

### Quick Start

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Asymptote-Labs/self-hosted.git
    cd self-hosted/terraform/aws
    ```
  </Step>
  <Step title="Configure variables">
    ```bash
    cp terraform.tfvars.example terraform.tfvars
    ```
    
    Edit `terraform.tfvars` with your settings:
    ```hcl
    aws_region     = "us-west-2"
    instance_type  = "t3.xlarge"
    asymptote_license_key = "your-license-key"
    ```
  </Step>
  <Step title="Deploy">
    ```bash
    terraform init
    terraform apply
    ```
  </Step>
  <Step title="Access Asymptote">
    After deployment completes, Terraform outputs the URL:
    ```
    asymptote_url = "https://asymptote.your-domain.com"
    ```
  </Step>
</Steps>

### What Gets Created

| Resource | Purpose |
|----------|---------|
| VPC | Isolated network with public/private subnets |
| EC2 | Application server running Docker Compose |
| RDS PostgreSQL | Managed database |
| ElastiCache Redis | Managed cache and queue |
| ALB | Load balancer with SSL termination |
| Route53 | DNS records (optional) |

## Manual Setup

For other cloud providers or on-premises deployments.

### Prerequisites

- Linux VM (Ubuntu 22.04 recommended)
- Docker and Docker Compose installed
- PostgreSQL 14+ database
- Redis 6+ instance
- Domain with SSL certificate

### Installation

<Steps>
  <Step title="Download configuration">
    ```bash
    curl -fsSL https://get.asymptotelabs.ai/docker-compose.yml -o docker-compose.yml
    curl -fsSL https://get.asymptotelabs.ai/.env.example -o .env
    ```
  </Step>
  <Step title="Configure environment">
    Edit `.env` with your settings:
    ```bash
    # Database
    DATABASE_URL=postgresql://user:pass@host:5432/asymptote
    
    # Redis
    REDIS_URL=redis://host:6379
    
    # License
    ASYMPTOTE_LICENSE_KEY=your-license-key
    
    # Security
    SECRET_KEY=generate-a-secure-random-string
    ```
  </Step>
  <Step title="Start services">
    ```bash
    docker compose up -d
    ```
  </Step>
  <Step title="Run migrations">
    ```bash
    docker compose exec app asymptote db migrate
    ```
  </Step>
  <Step title="Verify installation">
    ```bash
    curl http://localhost:8080/health
    ```
  </Step>
</Steps>

### Reverse Proxy Configuration

<Tabs>
  <Tab title="Nginx">
    ```nginx
    server {
        listen 443 ssl http2;
        server_name asymptote.your-domain.com;
        
        ssl_certificate /etc/ssl/certs/asymptote.crt;
        ssl_certificate_key /etc/ssl/private/asymptote.key;
        
        location / {
            proxy_pass http://localhost:8080;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
    ```
  </Tab>
  <Tab title="Caddy">
    ```
    asymptote.your-domain.com {
        reverse_proxy localhost:8080
    }
    ```
  </Tab>
</Tabs>

## Configuration

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `REDIS_URL` | Yes | Redis connection string |
| `ASYMPTOTE_LICENSE_KEY` | Yes | Your license key |
| `SECRET_KEY` | Yes | Secret for signing tokens |
| `GITHUB_APP_ID` | No | GitHub App ID for integrations |
| `GITHUB_APP_PRIVATE_KEY` | No | GitHub App private key |

### Scaling

For higher throughput, scale the worker containers:

```bash
docker compose up -d --scale worker=4
```

## Upgrades

To upgrade to a new version:

```bash
# Pull latest images
docker compose pull

# Restart with new version
docker compose up -d

# Run any new migrations
docker compose exec app asymptote db migrate
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Container fails to start">
    Check logs for errors:
    ```bash
    docker compose logs app
    ```
    
    Common issues:
    - Database connection string incorrect
    - Redis not reachable
    - Invalid license key
  </Accordion>
  <Accordion title="Database connection errors">
    Verify PostgreSQL is accessible:
    ```bash
    docker compose exec app pg_isready -h your-db-host
    ```
  </Accordion>
  <Accordion title="Out of memory">
    Increase VM memory or tune container limits in `docker-compose.yml`:
    ```yaml
    services:
      app:
        deploy:
          resources:
            limits:
              memory: 4G
    ```
  </Accordion>
</AccordionGroup>
