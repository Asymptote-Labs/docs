## Overview

Projects in Asymptote represent the GitHub repositories you want to monitor and protect. Each project gets its own security policy configuration, findings feed, and reporting.

## Steps

<Steps>
  <Step title="Open the Projects Page">
    From the Asymptote dashboard, navigate to **Projects** and click **Add Project**.
  </Step>
  <Step title="Select a Repository">
    Choose a repository from the list of connected GitHub repositories. If you don't see the repository you're looking for, make sure it's included in your [GitHub connection](/onboarding/connecting-github).
  </Step>
  <Step title="Configure the Project">
    Set a display name and optionally assign the project to a team. You can also select which security policy packs to apply, such as OWASP Top 10 or CWE Top 25.
  </Step>
  <Step title="Save and Activate">
    Click **Save**. Asymptote will immediately begin monitoring new pull requests for that repository.
  </Step>
</Steps>

## Managing Projects

You can view and manage all your projects from the **Projects** page. From there you can:

- **Enable or disable** monitoring for any project
- **Update policy assignments** at any time
- **Remove a project** if you no longer want Asymptote to monitor it

<Info icon="circle-info" title="Multiple Projects">
  You can add as many projects as you need. Each project is monitored independently with its own policy and findings history.
</Info>
