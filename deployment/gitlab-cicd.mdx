# GitLab CI/CD

Integrate Asymptote with GitLab CI/CD to automatically scan your code on every commit and merge request.

## Quick Start

Add the following to your `.gitlab-ci.yml`:

```yaml
asymptote-scan:
  image: asymptote/scanner:latest
  stage: test
  script:
    - asymptote scan --ci
  variables:
    ASYMPTOTE_API_KEY: $ASYMPTOTE_API_KEY
```

## Full Configuration

```yaml
stages:
  - test
  - security

asymptote-scan:
  image: asymptote/scanner:latest
  stage: security
  script:
    - asymptote scan --ci --severity critical,high --format sarif --output gl-sast-report.json
  variables:
    ASYMPTOTE_API_KEY: $ASYMPTOTE_API_KEY
  artifacts:
    reports:
      sast: gl-sast-report.json
    paths:
      - gl-sast-report.json
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
    - if: $CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH
```

## Configuration Options

### Environment Variables

| Variable | Description | Required |
|----------|-------------|----------|
| `ASYMPTOTE_API_KEY` | API key for authentication | Yes |
| `ASYMPTOTE_CONFIG` | Path to config file | No |
| `ASYMPTOTE_SEVERITY` | Severity filter | No |

### CLI Options

| Option | Description |
|--------|-------------|
| `--ci` | Enable CI mode |
| `--severity` | Filter by severity |
| `--format` | Output format |
| `--fail-on` | Fail threshold |

## Merge Request Integration

Enable security findings in merge request widgets:

```yaml
asymptote-scan:
  image: asymptote/scanner:latest
  stage: security
  script:
    - asymptote scan --ci --format gitlab-sast
  artifacts:
    reports:
      sast: gl-sast-report.json
```

## Security Dashboard

Results will appear in GitLab's Security Dashboard when using the SAST report format.

## Pipeline Templates

Use GitLab's include feature to share configurations:

```yaml
include:
  - remote: 'https://raw.githubusercontent.com/asymptote/templates/main/gitlab-ci.yml'

asymptote-scan:
  extends: .asymptote-template
  variables:
    ASYMPTOTE_SEVERITY: critical,high
```
