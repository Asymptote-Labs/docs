## Prerequisites

- An Asymptote account ([sign up here](https://asymptotelabs.ai/login))
- GitHub connected to Asymptote. See [Connecting GitHub](/onboarding/connecting-github) for setup steps.
- Admin access to the GitHub repositories you want to scan

For a full list of permissions Asymptote requests from GitHub, see the [GitHub integration page](/deployment/github).

## Overview

Asymptote now runs GitHub PR reviews through the Asymptote GitHub App. When connected, Asymptote scans pull requests, enforces security [policies](/introduction/core-concepts#policy), posts inline review comments, and updates a [check](/introduction/core-concepts#checks) run directly on the pull request.

## Connecting GitHub PR Reviews

Install and link the Asymptote GitHub App to the repositories you want reviewed. Once linked, pull request reviews are triggered automatically when a PR is opened or updated.

<Steps>
  <Step title="Install the GitHub App">
    From the Asymptote dashboard, go to **Settings → GitHub** and click **Install GitHub App**.

    Select the GitHub organization or account you want to connect, then authorize the Asymptote GitHub App.
  </Step>
  <Step title="Select repositories">
    Choose the repositories that Asymptote should review. You can grant access to all repositories in an organization or select specific repositories.

    Once the repository is linked in Asymptote, the GitHub App is onboarded automatically for PR reviews.
  </Step>
  <Step title="Open or update a pull request">
    Asymptote automatically reviews pull requests for linked repositories. On each PR open or update, Asymptote:

    - Reviews the full diff against your active security [policies](/introduction/core-concepts#policy)
    - Posts inline review comments with remediation guidance
    - Creates or updates the **Asymptote Security Review** check run
  </Step>
</Steps>

## Status Checks

Once connected, Asymptote posts status checks directly on your pull requests. You can configure branch protection rules to require the **Asymptote Security Review** check to pass before merging.

## Migrating from the GitHub Action

If your repository was previously configured with the legacy GitHub Action flow, migrate it to the GitHub App flow from the Asymptote dashboard:

<Steps>
  <Step title="Remove the repository in Asymptote">
    In the Asymptote dashboard, remove the repository from your connected repositories list.
  </Step>
  <Step title="Re-link the repository">
    Re-link the same repository through the GitHub App connection flow in the dashboard.
  </Step>
  <Step title="Verify automatic onboarding">
    The GitHub App should be onboarded automatically when the repository is linked again. After migration, the old GitHub Action workflow and `ASYMPTOTE_API_KEY` secret are no longer required for PR reviews.
  </Step>
</Steps>

## Troubleshooting

**Scans not triggering**
- Verify the GitHub App is installed on the target repository.
- Confirm the repository is linked in the Asymptote dashboard.
- Check that webhook deliveries are succeeding under your GitHub App settings → **Recent Deliveries**.

**Status checks stuck as pending**
- Large pull requests may take a few extra minutes to process.
- Re-link the repository from the Asymptote dashboard if it was originally connected through the legacy GitHub Action flow.

**Permission errors**
- Re-install the GitHub App and ensure the correct repositories are selected.
