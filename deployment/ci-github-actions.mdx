## Overview

Asymptote integrates with GitHub Actions to run security analysis as part of your CI pipeline. When connected, Asymptote scans pull requests, enforces security policies, and blocks deployments based on configurable severity thresholds — all without leaving your existing workflow.

## Required GitHub Permissions

When you install the Asymptote GitHub App, the following permissions are requested:

| Permission | Access | Reason |
|---|---|---|
| Administration, Issues, Metadata | Read | Enables Asymptote to monitor repository creation or deletion events, and access repository metadata. |
| Actions, Checks, Pull Requests, Webhooks, Workflows | Read & Write | Enables Asymptote to trigger workflows, create and update PR checks, create and update pull requests, and modify workflow files. |
| Contents | Read & Write | Enables Asymptote to detect vulnerabilities in code and open remediation PRs in a new branch. |
| Members | Read | Enables Asymptote to list and map teams and membership. |
| Deployments | Read & Write | Enables Asymptote to run security requirements on new deployments and block deployments based on security findings. |

## Connecting GitHub Actions

<Steps>
  <Step title="Install the GitHub App">
    From the Asymptote dashboard, navigate to **Settings → GitHub** and click **Install GitHub App**. Select your GitHub organization and choose which repositories to grant access to.
  </Step>
  <Step title="Add your API key">
    In your GitHub repository, go to **Settings → Secrets and variables → Actions** and add a new secret named `ASYMPTOTE_API_KEY` with your Asymptote API key.
  </Step>
  <Step title="Add the workflow">
    Create a workflow file in your repository to trigger Asymptote scans on pull requests.
  </Step>
</Steps>

## Example Workflow

```yaml
name: Asymptote Security Scan

on:
  pull_request:
    branches: [main, master]

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Run Asymptote Security Scan
        env:
          ASYMPTOTE_API_KEY: ${{ secrets.ASYMPTOTE_API_KEY }}
        run: |
          npx asymptote scan --repo ${{ github.repository }} --pr ${{ github.event.pull_request.number }}
```

## Status Checks

Once connected, Asymptote posts status checks directly on your pull requests. You can configure branch protection rules to require these checks to pass before merging.

### Severity Thresholds

| Setting | Behavior |
|---|---|
| **Strict** | Fails on Medium severity and above |
| **Standard** | Fails on High severity and above (default) |
| **Relaxed** | Fails only on Critical severity |

To adjust the threshold, go to **Project Settings → GitHub** in the Asymptote dashboard.

## Troubleshooting

**Scans not triggering**
- Verify the GitHub App is installed on the target repository.
- Check that webhook deliveries are succeeding under your GitHub App settings → **Recent Deliveries**.

**Status checks stuck as pending**
- Confirm your `ASYMPTOTE_API_KEY` secret is set correctly.
- Large pull requests may take a few extra minutes to process.

**Permission errors**
- Re-install the GitHub App and ensure the correct repositories are selected.
